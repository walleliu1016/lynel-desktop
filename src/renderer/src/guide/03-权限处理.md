# 权限处理

Claude 执行敏感操作（运行命令、写文件等）前会请求授权。Lynel Desktop 把权限请求同步到多个入口，方便你在任何地方处理。

## 权限请求出现时

当会话进入"等待授权"状态：

- 左侧会话列表的状态标签变为**等待授权**；
- 会话标签页图标变为警示样式；
- 灵动岛（屏幕顶部的悬浮胶囊）弹出提示；
- 如果配置了企业微信通道，会同时推送一张交互式权限卡片。

![等待授权状态](guide-images/06-awaiting-permission.png)

## 三种处理方式

任选其一处理，其余入口会自动关闭：

### 1. 终端内处理

像在命令行使用 Claude 一样，用方向键选择允许/拒绝后回车。

### 2. 灵动岛处理

鼠标移入灵动岛展开面板，点击允许或拒绝按钮。Claude 的提问（选择题）也可以直接在面板中作答。

![灵动岛权限面板](guide-images/07-notch-permission.png)

### 3. 企业微信处理

企业微信中会收到交互式模板卡片：

- **普通权限请求**：`button_interaction` 卡片，直接点击 **允许** / **拒绝** 按钮即可审批。
- **选择题（单选/多选）**：`vote_interaction` 卡片，点击选项后提交。
- **多个问题**：逐题发送卡片，全部答完后自动回复 Claude。

所有卡片点击后自动更新状态，无需输入命令。卡片按钮失效时（如网络问题），也可使用命令 `/pending` 查看待处理请求。

> 在终端里自行处理后，企业微信会收到"权限已在终端处理"的通知，会话状态同步恢复为运行中。
