# 企业微信集成

配置企业微信通道后，可以在手机上远程查看 Claude 的工作进度、发送新任务、处理权限请求。

## 配置通道

1. 打开 设置 → 通道，点击"新增通道"，选择企业微信。
2. 填写三项信息：
   - **chatId**：单聊填 userid，群聊填 chatid；
   - **botId**：企业微信机器人 ID；
   - **Secret**：企业微信机器人密钥。
3. 保存并启用，Claude 的对话进展会开始推送到对应聊天。

> **注意**：配置完成后，需要先在企业微信中主动与 Bot 对话一次（例如发送 `/help`），否则 Bot 无法向你推送消息。

![通道设置](guide-images/08-channel-settings.png)

## 常用命令

在企业微信聊天中发送以下命令（会话序号见 `/list`）：

| 命令 | 说明 |
|------|------|
| `/list` | 查看会话列表及序号 |
| `/bind <序号或sessionId>` | 绑定会话到当前聊天，之后直接发消息即发给 Claude |
| `/create <工作目录> [提示词]` | 远程创建新会话，桌面端会自动打开 |
| `/to <序号> <消息>` | 不绑定，临时向指定会话发一条消息 |
| `/allow <序号>` / `/deny <序号>` | 批准 / 拒绝该会话的权限请求（省略序号则处理最近一条） |
| `/answer <序号> <答案>` | 回答 Claude 的提问，如 `/answer 3 1,2` |
| `/pending` | 查看所有待处理的权限与提问 |
| `/status` | 查看当前绑定状态 |
| `/help` | 完整命令帮助 |

![企业微信对话示例](guide-images/09-wecom-chat.png)

## 消息推送内容

绑定后企业微信会收到：你发出的提示词、Claude 的工具调用（含命令/文件路径）、回复全文、权限请求卡片和会话结束通知。每条消息头部标注了项目名、会话序号和轮次，便于区分多个会话。

## 完整使用样例

下面是一次典型的远程协作流程。

**第一步：查看并绑定会话**

```
你：/list
机器人：可用会话列表
        | 序号 | 会话ID | 项目 |
        | 1 | a1b2c3d4 | my-project |
        | 2 | e5f6a7b8 | ease-ui |

你：/bind 2
机器人：已绑定会话
        | 会话ID | e5f6a7b8 |
        | 工作目录 | G:\work\ease-ui |
```

![企业微信绑定会话截图](guide-images/10-wecom-bind.png)

**第二步：直接发消息给 Claude**

绑定后，普通消息会直接发给该会话的 Claude：

```
你：帮我检查一下项目里有没有硬编码的颜色值
```

**第三步：接收工作进展推送**

```
机器人：> ease-ui · 会话#2 · 第3轮 · e5f6a7b8
        🔧 工具调用: Grep
        pattern: #[0-9a-fA-F]{6}

机器人：> ease-ui · 会话#2 · 第3轮 · e5f6a7b8
        🤖 Claude
        检查完成，发现 2 处硬编码颜色，已改为主题变量……
```

![企业微信接收工作进展截图](guide-images/11-wecom-progress.png)

**第四步：处理权限请求**

Claude 需要授权时会推送卡片：

```
机器人：> ease-ui · 会话#2 · 第4轮 · e5f6a7b8
        权限请求：Bash
        npm run test:main
        操作：/allow 2 或 /deny 2

你：/allow 2
机器人：已批准 会话#2 的权限请求
```

![企业微信处理权限请求截图](guide-images/12-wecom-allow.png)

**遇到 Claude 提问时**

```
机器人：Claude 向你提问：
        1. 选择实现方案
           1. 方案A：最小改动
           2. 方案B：完整重构
        回复示例：
        - /answer 2 1  单选

你：/answer 2 1
机器人：已提交 会话#2 的回答
```

![企业微信回答提问截图](guide-images/13-wecom-answer.png)

> 如果你在电脑前直接在终端里处理了权限，企业微信会收到"✅ 权限已在终端处理"，无需重复操作。
