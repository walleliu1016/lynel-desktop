# 企业微信集成

配置企业微信通道后，可以在手机上远程查看 Claude 的工作进度、发送新任务、处理权限请求。

## 创建企业微信机器人

在配置通道之前，需要先在企业微信中创建一个智能机器人：

1. 进入机器人管理页面，创建一个新的智能机器人。

![创建企业微信机器人](guide-images/15-wecom-create-bot.png)

2. 编辑机器人，API配置，使用长链接，创建完成后记录两项信息，配置通道时会用到：
   - **Bot ID**：机器人的唯一标识；
   - **Secret**：机器人的调用密钥（注意保密，不要泄露）。
![获取 Bot ID 与 Secret](guide-images/16-wecom-bot-credentials.png)
3. 获取 **chatId**：
   - **单聊**：填写你自己的企业微信 userid（可在管理后台通讯录中查看）；
   - **群聊**：把机器人添加进目标群，使用群的 chatid。

## 配置通道

1. 打开 设置 → 通道，点击"新增通道"，选择企业微信。
2. 填写三项信息：
   - **chatId**：单聊填 userid，群聊填 chatid；
   - **botId**：企业微信机器人 ID；
   - **Secret**：企业微信机器人密钥。
3. 保存并启用，Claude 的对话进展会开始推送到对应聊天。

> **注意**：配置完成后，需要先在企业微信中主动与 Bot 对话一次（例如发送 `/help`），否则 Bot 无法向你推送消息。

![通道设置](guide-images/08-channel-settings.png)

## 常用命令

在企业微信聊天中发送以下命令（会话序号见 `/list`）：

| 命令 | 说明 |
|------|------|
| `/list` | 查看会话列表及序号 |
| `/bind <序号或sessionId>` / `/switch <序号或sessionId>` | 绑定 / 切换当前聊天关联的会话 |
| `/unbind` / `/close` | 解绑当前聊天与会话的关联 |
| `/create <工作目录> [提示词]` / `/new ...` | 远程创建并绑定新会话 |
| `/to <序号> <消息>` | 不绑定，临时向指定会话发一条消息 |
| `/allow [序号或sessionId]` / `/y` / `/允许` | 批准权限请求（省略参数则处理最近一条） |
| `/deny [序号或sessionId]` / `/n` / `/拒绝` | 拒绝权限请求（省略参数则处理最近一条） |
| `/answer <序号> <答案>` / `/回答` | 回答 Claude 的提问，如 `/answer 3 1,2` |
| `/pending` | 查看所有待处理的权限与提问 |
| `/status` | 查看当前绑定状态 |
| `/help` | 完整命令帮助 |

## 权限请求与提问（模板卡片）

Claude 需要授权或提问时，企业微信会收到**交互式卡片**，点击按钮即可处理，无需输入命令。

### 普通权限请求

Claude 执行 Bash/Write/Read 等操作前，企业微信会收到一张 `button_interaction` 卡片：

- **来源**显示为 `项目：会话标题`，标题为"权限请求"
- 卡片主体展示工具名称与会话序号
- 点击 **允许** 或 **拒绝** 按钮直接审批
- 处理后卡片自动更新为"已批准该权限请求"/"已拒绝该权限请求"
- 在电脑终端自行处理权限后，企业微信会收到"✅ 权限已在终端处理"通知

![企业微信权限请求卡片](guide-images/17-wecom-permission-card.png)

### 选择题（单选/多选）

Claude 调用 AskUserQuestion 时，企业微信会收到 `vote_interaction` 卡片：

- 单选（mode=0）：radio 点选一个选项
- 多选（mode=1）：checkbox 勾选多个选项
- 点击 **提交** 按钮发送答案
- 提交后卡片更新为"已选择"状态

![企业微信投票卡片](guide-images/18-wecom-vote-card.png)

### 多个问题逐题发送

Claude 一次提出多个问题时，企业微信会采用逐题发送模式：

1. 先发一条文字预告，列出所有问题与选项。
2. 逐一发送投票卡片，每次只展示一个问题（标题标注"问题 1/3"等）。
3. 提交当前卡片后，自动发送下一张。
4. 全部答完后回复汇总消息：
   ```
   已收集全部回答，已回复 Claude：
   - 选择框架：Vitest
   - 安装依赖（多选）：eslint、prettier
   ```

![企业微信多问题汇总](guide-images/19-wecom-multi-question-summary.png)

> 所有卡片反馈消息都带有会话头部（`> **项目名** · 会话#N · \`xxxxxxxx\``），方便在多会话场景下区分。

## 引用消息快捷回复

不需要 `/to` 命令就能向指定会话发消息：

1. 长按机器人发送的任意消息，选择"引用"。
2. 直接输入要发送给 Claude 的内容。
3. 系统自动从被引用消息的头部解析会话，将内容转发到该会话。

引用消息中的会话识别来源是每条机器人消息自带的头部信息（`> **项目名** · 会话#N · \`xxxxxxxx\``），对于群聊中有多个会话交错时特别方便。

![企业微信引用消息路由](guide-images/20-wecom-quote-reply.png)

## 消息推送内容

绑定后企业微信会收到：你发出的提示词、Claude 的工具调用（含命令/文件路径）、回复全文、权限请求卡片和会话结束通知。每条消息头部标注了项目名、会话序号和 session ID，便于区分多个会话。

## 完整使用样例

下面是一次典型的远程协作流程。

**第一步：查看并绑定会话**

```
你：/list
机器人：可用会话列表
        | 序号 | 会话ID | 项目 |
        | 1 | a1b2c3d4 | my-project |
        | 2 | e5f6a7b8 | ease-ui |

你：/bind 2
机器人：已绑定会话
        | 会话ID | e5f6a7b8 |
        | 工作目录 | G:\work\ease-ui |
```

![企业微信绑定会话截图](guide-images/10-wecom-bind.png)

**第二步：直接发消息给 Claude**

绑定后，普通消息会直接发给该会话的 Claude：

```
你：帮我检查一下项目里有没有硬编码的颜色值
```

**第三步：接收工作进展推送**

```
机器人：> ease-ui · 会话#2 · e5f6a7b8
        🔧 工具调用: Grep
        pattern: #[0-9a-fA-F]{6}

机器人：> ease-ui · 会话#2 · e5f6a7b8
        🤖 Claude
        检查完成，发现 2 处硬编码颜色，已改为主题变量……
```

![企业微信接收工作进展截图](guide-images/11-wecom-progress.png)

**第四步：处理权限请求**

Claude 需要授权时会推送卡片：

```
机器人：> ease-ui · 会话#2 · e5f6a7b8
        权限请求：Bash
        npm run test:main
        操作：/allow 2 或 /deny 2

你：/allow 2
机器人：已批准 会话#2 的权限请求
```

![企业微信处理权限请求截图](guide-images/12-wecom-allow.png)

**遇到 Claude 提问时**

```
机器人：Claude 向你提问：
        1. 选择实现方案
           1. 方案A：最小改动
           2. 方案B：完整重构
        回复示例：
        - /answer 2 1  单选

你：/answer 2 1
机器人：已提交 会话#2 的回答
```

![企业微信回答提问截图](guide-images/13-wecom-answer.png)

> 如果你在电脑前直接在终端里处理了权限，企业微信会收到"✅ 权限已在终端处理"，无需重复操作。
