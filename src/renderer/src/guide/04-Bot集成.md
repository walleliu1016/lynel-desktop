# 企业微信集成

配置企业微信 Bot 后，可以在手机上远程查看 Claude 的工作进度、发送新任务、处理权限请求。企业微信集成依赖 Bot 系统，需要先配置 Bot 再绑定会话。

## 创建企业微信机器人

在配置之前，需要先在企业微信中创建一个智能机器人：

1. 进入机器人管理页面，创建一个新的智能机器人。

   ![创建企业微信机器人](guide-images/15-wecom-create-bot.png)

2. 编辑机器人，API 配置，使用长链接，创建完成后记录两项信息：
   - **Bot ID**：机器人的唯一标识；
   - **Secret**：机器人的调用密钥（注意保密，不要泄露）。

   ![获取 Bot ID 与 Secret](guide-images/16-wecom-bot-credentials.png)

3. 获取 **chatId**：
   - **单聊**：填写你自己的企业微信 userid（可在管理后台通讯录中查看）；
   - **群聊**：把机器人添加进目标群，使用群的 chatid。

> 每创建一个企业微信机器人，就可以绑定一个 Claude 会话。支持配置多个机器人，分别绑定不同的会话。

## 配置 Bot

企业微信集成依赖 Bot 系统。在连接企业微信之前，需要先配置 Bot。

### 1. 添加 Bot

打开 **设置 → 机器人管理**，点击"添加 Bot"，填写：

- **名称**：任意标识名，如"我的助手"
- **来源**：选择"企业微信"
- **Bot ID**：企业微信机器人 ID
- **Secret**：企业微信机器人 Secret

保存后 Bot 会自动连接。

![机器人管理](guide-images/21-bot-management.png)

### 2. 绑定会话

添加 Bot 后，需要将其绑定到一个 Claude 会话。有两种方式：

**方式一：新建会话时绑定**

在新建会话对话框中，选择"绑定机器人"下拉框，选中要绑定的 Bot。

![新建会话绑定机器人](guide-images/22-new-session-bind-bot.png)

**方式二：在已有会话中绑定**

在左侧会话列表中，右键点击目标会话，选择菜单中的 **绑定 Bot**，在弹出的列表中选择要绑定的 Bot。

![右键绑定机器人](guide-images/23-session-context-bind-bot.png)

绑定完成后，该会话的消息会自动推送到对应的企业微信聊天。一个 Bot 只能绑定一个会话，一个会话也只能绑定一个 Bot。

### 3. 解绑

- 右键已绑定的会话，选择 **解除绑定** 即可解绑。
- 在 **设置 → 机器人管理** 中删除 Bot，会自动解绑其关联的会话。

### 推送设置

在 **设置 → 通用设置** 中可以控制推送到企业微信的内容：

| 开关 | 说明 |
|------|------|
| 推送思考过程 | 关闭后不推送 Claude 的思考过程（thinking） |
| 推送工具调用 | 关闭后不推送工具调用消息（如 Bash 命令、文件读写等） |

## 权限请求与提问（模板卡片）

Claude 需要授权或提问时，企业微信会收到**交互式卡片**，点击按钮即可处理，无需输入命令。

### 普通权限请求

Claude 执行 Bash/Write/Read 等操作前，企业微信会收到一张 `button_interaction` 卡片：

- **来源**显示为项目名称（会话标题）
- 卡片主体展示工具名称与操作详情
- 点击 **允许** 或 **拒绝** 按钮直接审批
- 处理后卡片自动更新为"已批准"/"已拒绝"
- 在电脑终端自行处理后，企业微信会收到"权限已在终端处理"通知

![企业微信权限请求卡片](guide-images/17-wecom-permission-card.png)

### 选择题（单选/多选）

Claude 调用 AskUserQuestion 时，企业微信会收到 `vote_interaction` 卡片：

- 单选：radio 点选一个选项
- 多选：checkbox 勾选多个选项
- 点击 **提交** 按钮发送答案
- 提交后卡片更新为"已选择"状态

![企业微信投票卡片](guide-images/18-wecom-vote-card.png)

### 多个问题逐题发送

Claude 一次提出多个问题时，企业微信会采用逐题发送模式：

1. 先发一条文字预告，列出所有问题与选项。
2. 逐一发送投票卡片，每次只展示一个问题（标题标注"问题 1/3"等）。
3. 提交当前卡片后，自动发送下一张。
4. 全部答完后回复汇总消息：
   ```
   已收集全部回答，已回复 Claude：
   - 选择框架：Vitest
   - 安装依赖（多选）：eslint、prettier
   ```

![企业微信多问题汇总](guide-images/19-wecom-multi-question-summary.png)

> 所有卡片和推送消息都带有会话头部（`> **项目名** · 会话#N · \`xxxxxxxx\``），在多会话场景下清晰区分。

## 引用消息快捷回复

不需要 `/to` 命令就能向指定会话发消息：

1. 长按机器人发送的任意消息，选择"引用"。
2. 直接输入要发送给 Claude 的内容。
3. 系统自动从被引用消息的头部解析会话，将内容转发到该会话。

每条机器人消息自带头部信息（`> **项目名** · 会话#N · \`xxxxxxxx\``），群聊中有多个会话交错时特别方便。

![企业微信引用消息路由](guide-images/20-wecom-quote-reply.png)

## 控制指令

在企业微信中发送以下命令可远程操控 Claude 终端进程（不转发给 Claude，仅在 PTY 层面生效）：

| 命令 | 效果 |
|------|------|
| `/interrupt` `/ctrl-c` `/ctrl+c` | 中断 Claude 当前生成（等同在终端按 Ctrl+C） |
| `/escape` `/esc` | 发送 Esc 键 |
| `/ctrl-d` | 发送 Ctrl+D（EOF） |
| `/ctrl-z` | 发送 Ctrl+Z（SIGTSTP） |

**使用场景：**
- Claude 陷入死循环或输出过长时，发送 `/interrupt` 可立即中断
- Claude 处于交互式模式时，发送 `/escape` 退出当前模式
- 需要结束标准输入时，发送 `/ctrl-d`

> 命令使用与普通消息相同的路由逻辑：引用消息中的会话头部自动识别目标会话，或通过 Bot 绑定关系路由到对应会话。执行成功后会收到"已发送 xxx"的确认回复。

## 消息推送内容

绑定后企业微信会收到：

- **会话启动通知**：机器人绑定后，会话启动时推送"项目 **xxx** 工作目录会话已启动"
- **你的提示词**：自动过滤 Claude 注入的 `<system-reminder>` 标签，只保留实际输入
- **Claude 的回复**：文本回复和思考过程（可在设置中关闭思考推送）
- **工具调用**：执行的命令、文件操作等（可在设置中关闭工具调用推送）
- **权限请求卡片**：允许/拒绝/选择题卡片交互
- **工具执行结果**：工具执行完成的通知

每条消息头部标注了项目名、会话序号和 session ID，便于区分多个会话。

## 完整使用样例

下面是一次典型的远程协作流程。

**第一步：直接发消息给 Claude**

绑定后，普通消息会直接发给该会话的 Claude：

```
你：帮我检查一下项目里有没有硬编码的颜色值
```

**第二步：接收工作进展推送**

```
机器人：> my-project · 会话#3 · x1y2z3w4
        🔧 工具调用: Grep
        pattern: #[0-9a-fA-F]{6}

机器人：> my-project · 会话#3 · x1y2z3w4
        🤖 Claude
        检查完成，发现 2 处硬编码颜色，已改为主题变量……
```

![企业微信接收工作进展截图](guide-images/11-wecom-progress.png)

**第三步：处理权限请求（卡片）**

Claude 需要授权时直接推送卡片，点击按钮即可，无需命令：

```
机器人：> my-project · 会话#3 · x1y2z3w4
        ┌──────────────────────────────┐
        │  权限请求                     │
        │  工具：Bash                   │
        │  npm run test:main           │
        │                              │
        │  [✅ 允许]  [❌ 拒绝]         │
        └──────────────────────────────┘
```

你在终端自行处理后，企业微信会收到"✅ 权限已在终端处理"，无需重复操作。
