# 常见问题

## 终端没有反应怎么办？

1. 查看会话状态：如果是"等待授权"，先处理权限请求（见"权限处理"章节）。
2. 确认消息以回车结尾——直接在终端输入后按回车即可。
3. 如果终端一直停留在加载状态，检查 设置 → 通用 中的 Claude 路径是否正确；留空表示使用系统 PATH 中的 `claude`。

## 点了关闭窗口，程序去哪了？

关闭按钮是**隐藏到托盘**，程序仍在后台运行，会话不会中断。点击系统托盘图标可以重新打开窗口；需要完全退出时使用托盘菜单。

## 历史会话打不开或对话丢失？

历史会话依赖 Claude 的会话记录文件。如果对应工作目录下的记录被清理，会话将无法恢复上下文，建议重新创建会话。

## 企业微信收不到推送？

1. **先与 Bot 对话一次**：配置完成后必须在企业微信中主动给 Bot 发一条消息（如 `/help`），否则 Bot 无法推送。这是最常见的原因。
2. 检查 设置 → 通道 中企业微信通道是否已启用，chatId / botId / Secret 是否填写正确。
3. 单聊与群聊的 chatId 格式不同：单聊填 userid，群聊填 chatid。
4. 通道推送失败不会影响 Claude 正常运行，可查看日志排查。

## 切换了模型供应商但没生效？

供应商配置在**新启动的会话**上生效。已在运行的会话仍使用启动时的配置，重新打开该会话即可切换到新供应商。

## 会话状态显示不准确？

会话状态由本地代理和 hook 事件驱动。如果长时间不同步，可尝试关闭并重新打开该会话标签页；仍异常时重启应用即可恢复。
