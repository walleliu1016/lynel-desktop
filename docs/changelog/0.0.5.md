## 0.0.5

### 灵动岛悬浮窗

- 新增 iOS 灵动岛风格常驻桌面悬浮窗（`src/main/notch-window.ts`），透明无边框、始终置顶、跳过任务栏
- 新增 `frontend/src/views/NotchView.vue` — 独立入口页面，支持闭口态药丸与开口态展开面板
- 螃蟹图标 5 色状态指示：空闲灰色、思考橙色、工作橙色、等待权限琥珀色、错误红色
- hover 展开 / leave 收起，展开后显示会话列表与活跃状态
- 会话列表重新设计：hover 反馈（translateX + 阴影）、强调色条、轮次 Badge
- 权限审批卡片：允许/拒绝按钮，支持 AskUserQuestion 问答交互（单选/多选/自定义输入）
- AskUserQuestion 动态高度适配，不再需要滚轮
- 修复展开收起后旧区域无法点击的问题（Windows 鼠标命中区域重算）

### 权限仲裁器

- 新增 `src/main/permission-broker.ts` — 主进程单例权限管理器
- 序号预分配（`allocateSeq`），在企业微信消息中显示 `#1` 而非 UUID
- `cancelBySessionTool`：终端自行解决权限时联动关闭灵动岛和企业微信审批 UI
- `onResolve` 通过 EventBus 同步状态到灵动岛渲染进程，确保跨通道审批后 UI 实时消失

### 企业微信权限审批

- 支持 PermissionRequest 卡片推送，展示 `#allow <seq>` / `#deny <seq>` 回复格式
- 修复 `plugin.outbound.sendMarkdown` 不存在导致推送失败的问题，改为 `sendText` 降级
- PermissionResolved 事件跨通道同步，企业微信审批后灵动岛自动关闭

### HookServer 增强

- 新增 `req.on('close')` 监听：终端侧自行解决权限（用户敲 y/n）时通知 broker 取消待审批请求
- PermissionRequest / PermissionResolved 事件通过 ChannelDispatcher 统一路由到各输出通道

### Channel 事件扩展

- 新增 `PermissionRequest`、`PermissionResolved` 事件类型
- ChannelDispatcher 支持事件级分发，各 channel 按需处理

### 前端改动

- `PermissionToast.vue`：新增 approve/deny 按钮，直接调用 broker 完成审批
- `useElectron.ts`：新增 `ResolvePermission`、`SetNotchSize`、`SetNotchPassthrough` IPC 方法
- Vite config：多入口构建，支持 notch 独立页面打包
