## 0.0.8

### 企业微信模板卡片交互

- PermissionRequest / AskUserQuestion 推送到企业微信为 `button_interaction` / `vote_interaction` 模板卡片，用户点击允许/拒绝/提交即可审批
- 多问题自动逐题发送：先发文字预告，再逐张发送投票卡片，全部答完后回复汇总列表
- `source.desc` 显示 `项目：会话标题` 标签，区分多个会话
- `/allow` `/deny` `/answer` 命令保留作为降级方案
- 卡片被处理后自动更新状态（已批准/已拒绝/已选择），重复点击提示"已被处理"

### 引用消息路由

- 在企业微信中引用机器人消息并输入内容，自动从引用消息的会话头部解析目标会话，无需 `/to` 命令
- 群聊中多个会话交错时特别方便，引用哪条消息就发到哪个会话
- `/to` 命令保留作为备用方式

### 统一消息头部

- 所有会话相关反馈消息（已批准/已拒绝/已选择/已收集等）统一带 `> **项目名** · 会话#N · \`sid\`` 头部
- 去掉消息中无意义的"第N轮"，只保留项目名、会话序号、session ID

### 使用指南

- 新增企业微信模板卡片交互、引用消息快捷回复章节与配图
- 权限处理指南更新为卡片审批优先于命令
- 右侧增加标题目录导航（TOC），自动解析 h1/h2/h3 标题，当前章节滚动高亮

### 界面

- 登录页设置按钮从顶部 TitleBar 移至底部版本号旁边，更简洁
