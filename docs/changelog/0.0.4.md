## 0.0.4

### 会话创建机制重构

- **改用 `--session-id` 预生成 UUID 创建会话**，不再依赖 `SessionStart` hook 返回 session ID，消除 hook 超时导致的会话创建失败问题
- 移除 `SessionStart` hook 的配置与脚本生成逻辑（`ensureSessionStartScript`、`session-start.sh/ps1`）
- 代理在会话创建时使用预生成的 UUID，不再需要先启动临时代理再迁移

### 通道系统重构

- **通道页面重新设计**：侧边栏 + 详情面板布局，支持在企业微信、飞书、本地文件三个通道间切换
- **互斥开关**：同时只能启用一个通道，切换时自动关闭其他通道
- 将企业微信配置抽取到独立的 `WeComConfig.vue` 组件
- 新增 `FeishuConfig.vue`（占位）和 `LocalFileConfig.vue` 组件
- 保存时一次写入所有通道配置，配置路径从 `wecom` 迁移到 `channels` 命名空间，兼容旧版配置

### 本地文件通道

- 新增 `LocalFileChannel` 实现 `OutputChannel` 接口，将阶段事件写入磁盘文件
- 默认输出路径：`~/.lynel-desktop/output/<project>/<sessionId>.jsonl`
- 支持 JSONL（每行一条记录）和 JSON（数组）两种格式
- 自动过滤流式 `text`/`thinking` 碎片，只保留 `prompt`、`tool_use`、`tool_result`、`response_complete` 完整事件
- 启动时自动加载已保存的通道配置

### 企业微信增强

- **工具调用显示详情**：`tool_use` 消息现在展示命令内容（Bash）、文件路径（Read/Write/Edit）、搜索模式（Grep/Glob）等具体参数
- 通过累积 Anthropic API 的 `input_json_delta` 增量解决流式模式下 `content_block_start` 输入为空的问题
- 新增 `#create` 命令支持

### API 代理修复

- **修复 `tool_result` 事件未被捕获**：代理现在从请求体中检测 `tool_result` 块并发射事件，携带 `tool_use_id` 与对应 `tool_use` 关联
- 流式 SSE 数据增量解析，避免响应末尾一次性处理大量数据阻塞事件循环
- `tool_use` 在 `content_block_stop` 或 `stop_reason: 'tool_use'` 时发射，确保 `input_json_delta` 已完全接收

### 其他修复

- 终端新增原生右键菜单（复制、粘贴、全选）
- 登录后窗口自动居中显示
