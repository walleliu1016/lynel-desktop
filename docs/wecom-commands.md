# 企业微信命令使用说明

在企业微信中向机器人发送以 `/` 开头的消息，可以管理本地 Claude 会话、指定消息发往哪个会话。

> 命令消息不会转发给 Claude，只在企业微信与 Lynel Desktop 之间处理。

## 命令速查

| 命令 | 说明 |
|---|---|
| `/list` | 列出当前所有本地会话 |
| `/create <工作目录> [提示词]` / `/new ...` | 创建并绑定新会话 |
| `/bind <id/序号>` / `/switch <id/序号>` | 将当前聊天绑定到指定会话 |
| `/to <id/序号> <消息>` | 单条消息临时发送到指定会话，不修改默认绑定 |
| `/status` | 查看当前聊天绑定的会话及状态 |
| `/unbind` / `/close` | 解绑当前聊天与会话的关联 |
| `/allow [id/序号]` / `/y` / `/允许` | 批准权限请求（省略参数则处理最近一条） |
| `/deny [id/序号]` / `/n` / `/拒绝` | 拒绝权限请求（省略参数则处理最近一条） |
| `/pending` | 查看当前待处理的权限请求 / Claude 提问 |
| `/answer <id/序号> <答案>` / `/回答` | 回答 Claude 的 AskUserQuestion 提问 |
| `/help` | 显示命令帮助 |

## 会话编号

`/list` 会按序号列出可用会话。命令中既可以用序号，也可以用 session ID（支持前缀自动补全）。

```
可用会话：
| 序号 | 会话ID | 状态 | 项目 |
| 1 | a1b2c3d4 | running | /home/user/project-a |
| 2 | e5f6g7h8 | done | /home/user/project-b |
```

## 常用流程

### 1. 查看会话

```
/list
```

### 2. 绑定到某个会话

```
/bind 1
```

绑定后，该聊天里发送的普通文字消息都会进入会话 `1` 对应的 Claude PTY。

### 3. 切换会话

```
/switch 2
```

### 4. 临时发到另一个会话

```
/to 2 帮我解释这段代码
```

这条消息只进入会话 `2`，默认绑定仍然保持不变。

### 5. 查看绑定状态

```
/status
```

### 6. 解绑

```
/unbind
```

解绑后，普通消息不再自动转发，直到重新 `/bind`。

### 7. 创建新会话

```
/create /home/user/project 帮我优化代码
```

创建成功后自动绑定到当前聊天。

### 8. 审批权限请求

```
/allow 3
/deny 3
```

或直接处理最近一条：

```
/y
/n
```

### 9. 回答 Claude 提问

```
/answer 3 1
/answer 3 1,2
/answer 3 我的自定义回答
```

## 路由规则

发送普通文字消息时，系统按以下优先级选择目标会话：

1. **命令匹配**：`/to` 等命令显式指定会话。
2. **引用消息路由**：消息引用了机器人之前发送的消息时，自动从引用内容中解析会话头部，直接路由到该会话。
3. **显式绑定**：该聊天通过 `/bind` / `/switch` 绑定过会话，且会话仍存在、工作目录未变更。
4. **最近活跃**：该聊天有过 outbound 消息记录（Claude 回复推送到企业微信）。
5. **最后活跃 session**：当前应用生命周期内最后产生过阶段事件的 session。
6. 都没有命中时，企业微信会收到提示，引导使用 `/list` / `/bind`。

### 引用消息快捷路由

引用机器人发送的任意消息（长按消息 → 引用），然后直接输入内容，系统会自动将消息转发到该消息所属的会话，无需 `/to` 命令。

- 系统会从被引用消息的头部（`> **项目名** · 会话#N · \`xxxxxxxx\``）中解析会话。
- 如果被引用的消息不是机器人发送的，或内容中找不到会话头部，则会回退到绑定/最近活跃路由。
- 引用路由结果与 `/to` 等效，不会修改当前绑定的会话。

## 模板卡片交互

权限请求和 Claude 提问在企业微信中以**模板卡片**形式呈现，点击按钮即可处理，无需输入命令。

### 卡片类型

| 场景 | 卡片类型 | 交互方式 |
|------|---------|---------|
| 普通权限请求（Bash/Write/Read） | `button_interaction` | 点击 **允许** 或 **拒绝** 按钮 |
| 单个选择题 | `vote_interaction` | 点选选项后点击 **提交** |
| 多个选择题 | 多张 `vote_interaction` | 逐张提交，全部答完后回复汇总 |

### 卡片来源标签

每张卡片的来源区域会显示 `项目：会话标题`，方便区分多个会话的请求。

### 多问题逐题回答

Claude 提出多个问题时，企业微信会：

1. 先发送一条文字预告，列出所有问题与选项。
2. 逐一发送投票卡片，一次只展示一个问题。
3. 每提交一张卡片，该卡片会更新为"已选择"状态，并发送下一张。
4. 全部答完后回复汇总消息，列出每题答案，确认已转发给 Claude。

### 已处理状态

- 卡片被处理后会自动更新为"已批准"/"已拒绝"/"已选择"等状态。
- 重复点击已处理的卡片会收到提示"该请求已被处理或已过期"。
- 在终端自行处理权限后，企业微信卡片不会被更新（终端是独立通道）。

## 注意事项

- 命令不区分大小写。
- 使用前缀匹配 session ID 时，如果命中多个会话，会提示使用完整 ID。
- 本地删除 session 后，相关绑定会自动清理，再次发送消息时会提示重新绑定。
- 绑定关系持久化在本地，重启 Lynel Desktop 后仍然有效。
