# 企业微信命令使用说明

在企业微信中向机器人发送以 `#` 开头的消息，可以管理本地 Claude 会话、指定消息发往哪个会话。

> 命令消息不会转发给 Claude，只在企业微信与 Lynel Desktop 之间处理。

## 命令速查

| 命令 | 说明 |
|---|---|
| `#list` | 列出当前所有本地会话 |
| `#bind <id/序号>` | 将当前聊天绑定到指定会话 |
| `#switch <id/序号>` | 同 `#bind`，用于切换已绑定会话 |
| `#to <id/序号> <消息>` | 单条消息临时发送到指定会话，不修改默认绑定 |
| `#status` | 查看当前聊天绑定的会话及状态 |
| `#unbind` / `#close` | 解绑当前聊天与会话的关联 |

## 会话编号

`#list` 会按 `1.`、`2.`、`3.` 的格式列出可用会话。命令中既可以用序号，也可以用 session ID（支持前缀自动补全）。

```
可用会话：
1. a1b2c3d4 [running] /home/user/project-a
2. e5f6g7h8 [done] /home/user/project-b
```

## 常用流程

### 1. 查看会话

```
#list
```

### 2. 绑定到某个会话

```
#bind 1
```

绑定后，该聊天里发送的普通文字消息都会进入会话 `1` 对应的 Claude PTY。

### 3. 切换会话

```
#switch 2
```

### 4. 临时发到另一个会话

```
#to 2 帮我解释这段代码
```

这条消息只进入会话 `2`，默认绑定仍然保持不变。

### 5. 查看绑定状态

```
#status
```

### 6. 解绑

```
#unbind
```

解绑后，普通消息不再自动转发，直到重新 `#bind`。

## 路由规则

发送普通文字消息时，系统按以下优先级选择目标会话：

1. **显式绑定**：该聊天通过 `#bind` / `#switch` 绑定过会话，且会话仍存在、工作目录未变更。
2. **最近活跃**：该聊天有过 outbound 消息记录（Claude 回复推送到企业微信）。
3. **最后活跃 session**：当前应用生命周期内最后产生过阶段事件的 session。
4. 都没有命中时，企业微信会收到提示，引导使用 `#list` / `#bind`。

## 注意事项

- 命令不区分大小写。
- 使用前缀匹配 session ID 时，如果命中多个会话，会提示使用完整 ID。
- 本地删除 session 后，相关绑定会自动清理，再次发送消息时会提示重新绑定。
- 绑定关系持久化在本地，重启 Lynel Desktop 后仍然有效。
