核心变更 vs 当前实现

维度 当前 新设计
会话发现 启动时扫描 ~/.lynel-desktop/projects/ 下所有 JSONL，展示全部历史会话 不扫描。用户主动"打开项目"才创建/恢复会话。历史记录存在本地 JSON 文件中
会话列表 标签页切换：所有 / 运行中 / 结束 + 搜索栏 单一扁平列表，只显示用户主动打开的项目。无标签页、无状态过滤
新建会话 弹出 NewSessionDialog：选目录 + 必填提示词 + 选项 → 创建 打开项目"对话框：选目录即可，提示词可选。打开 = 新建，无需区分"新建"和"打开"
主页 直接进入 HomeView，无活跃会话时显示空状态提示 欢迎页：Logo + 打开项目按钮 + 最近项目列表。类似 VS Code 的 Start 页面
历史持久化 隐式：通过扫描 projects 目录获得 显式：打开过的项目写入 recent.json，最多保留 50 条，按最近打开时间排序
关闭会话 无明确关闭操作，状态变为 ended/done 后仍在列表中 侧边栏每个会话有 × 按钮，点击关闭 → 停止 Claude 进程 → 从侧边栏移除（不删除 JSONL 数据）

实施计划

1 新增 WelcomeView.vue — 欢迎页面组件（Logo + 打开项目按钮 + 最近项目列表）
2 新增 recent store — 管理最近打开的项目列表，持久化到 ~/.lynel-desktop/recent.json
3 重写 SessionList.vue — 去掉标签页/搜索，改为简洁的已打开会话列表
4 重写 NewSessionDialog → OpenProjectDialog — 简化为选目录 + 可选提示词
5 调整路由 — 登录后进入 /welcome；打开项目后进入 /home（工作区视图）
6 简化 sessions store — 移除 adopted / opened 等冗余状态；后端不再扫描 projects 目录
